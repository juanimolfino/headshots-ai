# Handoff: headshotly.pro — Landing Page + Design System

## Overview
This package contains the **headshotly.pro** marketing landing page, designed as a single
HTML reference file. Beyond the one page, it defines a small, reusable **design system**
(colors, type, spacing, components) that should become the visual foundation for the rest
of the app's frontend.

**Two jobs for Claude Code:**
1. Implement this landing page at `app/(marketing)/page.tsx` (or `app/page.tsx`).
2. Adopt the design tokens + primitives below across the existing frontend so the whole
   product matches this look.

## About the Design Files
The file in this bundle (`headshotly.pro Landing Page.html`) is a **design reference
created in HTML** — a prototype showing the intended look and behavior. It is **not**
production code to copy verbatim. Recreate it in the target stack (Next.js App Router +
Tailwind + shadcn/ui + TypeScript) using that stack's idioms (`next/font`, `next/image`,
server components, shadcn primitives). Match the visuals pixel-for-pixel; rebuild the
structure idiomatically.

## Fidelity
**High-fidelity.** Colors, typography, spacing, and interactions are final. Reproduce the
UI pixel-perfectly using the codebase's libraries. Exact values are listed under
**Design Tokens** below.

No gradients, no glassmorphism, no heavy shadows. Shadows are used only as faint
ambient lifts (e.g. `0 30px 70px -40px rgba(20,27,50,.85)` on the Pro card).

### Typography
- **Display (serif):** Newsreader — weights 400/500/600, optical sizing on. Used for
  `h1`/`h2`/`h3`, prices, FAQ questions, footer tagline (italic).
- **Body / UI (sans):** Hanken Grotesk — weights 400/500/600/700.
- **Mono:** system mono (`ui-monospace`) for placeholder labels and small tags.

Load with `next/font` (no render-blocking CDN):
```ts
// app/fonts.ts
import { Newsreader, Hanken_Grotesk } from "next/font/google";
export const serif = Newsreader({ subsets: ["latin"], weight: ["400","500","600"], variable: "--font-serif", display: "swap" });
export const sans  = Hanken_Grotesk({ subsets: ["latin"], weight: ["400","500","600","700"], variable: "--font-sans", display: "swap" });
```
Apply `${serif.variable} ${sans.variable}` to `<html>`; set `font-family` via the CSS vars.

**Type scale (clamp-based, responsive):**
| Element | Size | Weight | Tracking |
|---|---|---|---|
| h1 (hero, single use) | `clamp(40px, 6.2vw, 72px)` | 500 | `-.025em` |
| h2 (section) | `clamp(30px, 4vw, 46px)` | 500 | `-.02em` |
| h3 (cards, steps, FAQ) | 20–23px | 500 | `-.01em` |
| Body | 15–19px | 400 | normal, `line-height: 1.6` |
| Eyebrow | 12.5px / `.14em` uppercase | 600 | with a 22px leading rule |

### Spacing & shape
- Container max-width: **1180px**, side padding 28px (22px on mobile).
- Section vertical padding: **96px** desktop, 72px mobile.
- Radii: buttons `999px` (pill); cards `14px`; plans `20px`; navy panel `26px`.
- Reveal motion: fade + 14px rise on scroll (IntersectionObserver), `.7s` ease
  `cubic-bezier(.2,.7,.2,1)`. Respect `prefers-reduced-motion`. **No hover-lift transforms**
  on cards (subtle motion only).

## Screens / Views — Landing Page sections

Single page, semantic landmarks: `nav` → `main` → `footer`. Section IDs:
`#how-it-works`, `#styles`, `#pricing`, `#faq`.

1. **Nav (sticky):** logo `headshotly.pro` (navy dot + serif wordmark) · links How it works /
   Styles / Pricing / FAQ · "Sign in" (ghost) + "Get started" (primary). Adds a hairline
   bottom border once `scrollY > 8`. Collapses to a hamburger + slide-down panel below 760px.
2. **Hero:** 2-col. Left: eyebrow, `h1` "Your professional headshot, *generated by AI*"
   (last 3 words italic navy), subhead, primary CTA "Get my headshots →", text link
   "See how it works ↓", social-proof row (4 stacked avatars + "Trusted by 1,200+
   professionals worldwide"). Right: **2×2 grid of 4 headshot placeholders**, cols 1 & 4
   nudged up 14px for rhythm.
3. **How it works:** section title "Three steps to a studio-quality headshot"; 3 steps
   (image + "Step 0N" + h3 + sentence), each with a top rule. Stacks on mobile.
4. **Styles:** centered head "Choose your look"; 3 cards (3:4 image, mono tag, h3, one line):
   Professional / Cinematic / Natural.
5. **Why not a photographer:** full-width **navy panel**, light text; head "Better than a
   photoshoot. Here's why."; 4 items (thin line icon + h3 + sentence): Cost, Speed,
   Variety, Comfort.
6. **Pricing:** centered head "Simple, one-time pricing. No subscriptions."; 2 plan cards

7. **FAQ:** centered head "Common questions"; 6-item single-open accordion. Use the native

8. **Footer:** CTA band "Your next headshot is ten minutes away." + primary CTA; brand +
   italic tagline "Studio-quality headshots, generated by AI."; links Privacy / Terms /
   Contact; "© 2026 headshotly.pro".

## Interactions & Behavior
- **Sticky nav:** toggle a `scrolled` class/state past 8px for the border.
- **Mobile menu:** hamburger toggles a slide-down panel; closes on link click; manage
  `aria-expanded`.
- **Scroll reveal:** IntersectionObserver adds an `in` state (fade + rise). Content must be
  visible without JS (above-the-fold renders server-side) — gate the *hidden* start state
  behind a `js` flag, not the content itself.
- **FAQ:** single-open accordion.
- **Smooth anchor scrolling** to section IDs.

## State Management
Minimal — all local/UI state: `navScrolled` (boolean), `mobileOpen` (boolean), accordion
open-item (string | null). No data fetching on this page.

## SEO requirements (carry over exactly)
